## 项目名称
**小澄觉察助手（Awareness Guide）**
*向外看的人在做梦，向内看的人在觉醒。*


## 项目简介（300字）

小澄是一个 AI 驱动的觉察引导系统，帮用户逐步找到并建立稳定的内核。

底层哲学是**减法疗愈**——不是塞更多道理，是帮用户看见遮盖住本有注意力的东西。理论三角：庄子（为什么要放下）→ 清静经（怎么放下）→ 王阳明（放下后怎么行动）。有了内核，才能物来顺应、不迎不拒。

不是管理时间，不是做计划，是建立内核。

核心差异化：不教冥想放松，不靠番茄钟强制，帮用户看清分心的因果链。前期以对话式觉察引导为主，不依赖硬件；后期可整合可穿戴设备数据（情绪感知辅助）。

10倍愿景：不按"独立开发者副业项目"做决策，按"下一代心理健康基础设施"的估值来设计架构。

技术实现上，复用 OpenClaw SKILL 框架 + 飞书 Bot，纯 Python 标准库实现，零外部依赖。JSON 文件存储，数据层与 AI 底座解耦，支持随时迁移到微信服务号独立部署。

## 项目亮点

### 1. 理论内核 —— 庄子→清静经→王阳明
- **庄子**：为什么要放下——"心斋""坐忘"，放下的是遮蔽，不是追求
- **清静经**：怎么放下——"人能常清静，天地悉皆归"，静下来的方法
- **王阳明**：放下后怎么行动——"知行合一""致良知"，有了内核后的行动力
- 不是知识灌输，是帮用户在觉察对话中**体验**到这些

### 2. 减法疗愈框架
```
看见因果链 → 放下遮蔽 → 建立内核 → 物来顺应
（觉察对话） （减法引导） （内核稳定） （自然行动）
```
不塞道理，帮用户自己看见。

### 3. AI 驱动觉察引导
- 对话引擎：SKILL 驱动，复刻 Anthropic Claude Code Agent Harness 8 步循环
- 自动化：晨间/晚间定时推送觉察引导
- 减法式对话：提问多于建议，帮用户看清分心因果链，而非说教

### 4. 零成本启动
- Python 标准库，零外部依赖
- JSON 文件存储，无需数据库
- 复用现有飞书 Bot，零渠道成本

## 市场定位

### 目标用户
泛 ADHD 人群——注意力碎片化、刷手机停不下来、知道该做但做不到的人。

### 用户痛点
- "我知道该做什么，但我就是做不到"——不是缺知识，是缺内核
- 刷手机停不下来——注意力被算法劫持，自己无能为力
- 信息过载导致焦虑——越看越慌，越慌越看
- 冥想/番茄钟/时间管理App都没用——问题不在工具，在没有内核

### 竞争差异
| | 冥想/正念 App | 番茄钟/时间管理 | 小澄觉察助手 |
|---|---|---|---|
| 核心方法 | 教你放松 | 强制专注 | 帮你看见分心因果链 |
| 理论基础 | 正念/冥想 | 行为主义 | 庄子→清静经→王阳明 |
| 体验设计 | 引导式呼吸 | 计时+打断 | 对话式觉察引导 |
| 核心目标 | 缓解焦虑 | 提高效率 | 建立稳定内核 |
| 用户感受 | "好放松" | "好有压迫感" | "原来我是这样分心的" |
| 硬件依赖 | 无 | 无 | 前期无，后期可穿戴 |
