# 觉察助手 · 微信服务号产品原型 v1

> 2026-05-17 | 小麦 🌾

---

## 一、为什么是服务号？

| 考量 | 服务号 | 小程序 | 独立App |
|------|--------|--------|---------|
| 开发成本 | ★☆☆ 最低 | ★★☆ | ★★★ |
| 推送能力 | ✅ 模板消息（4条/月付费可更多） | ❌ 无主动推送 | ✅ 但需下载 |
| 用户门槛 | 关注即用 | 需搜索/扫码 | 需下载注册 |
| 对话体验 | ✅ 原生聊天界面 | 需自建UI | 需自建UI |
| MVP速度 | 🚀 最快 | 中等 | 最慢 |

**结论**：MVP 用服务号。用户关注后直接对话，零学习成本。后续可嵌小程序做可视化面板（PERMA趋势图等）。

## 二、交互架构

### 2.1 底部菜单（3个）

```
┌──────────────┬──────────────┬──────────────┐
│   🌅 觉察     │   📊 我的     │   ❓ 关于     │
└──────────────┴──────────────┴──────────────┘
```

| 菜单 | 点击行为 | 说明 |
|------|---------|------|
| 🌅 觉察 | 发送文本「开始觉察」 | 触发一次即时觉察对话 |
| 📊 我的 | 打开 H5 页面（或跳小程序） | PERMA趋势图、streak、里程碑、品格画像 |
| ❓ 关于 | 发送图文消息 | 产品介绍、使用指南、隐私政策 |

### 2.2 主动推送规则

| 时段 | 触发条件 | 推送内容 | 频率控制 |
|------|---------|---------|---------|
| **晨间** 7:00-7:30 | 用户曾完成晚间觉察 ≥1次 | 晨间觉察引导（Round 1） | 1次/天 |
| **晚间** 21:30-22:00 | 用户曾完成晨间觉察 ≥1次 | 晚间觉察引导（Round 1） | 1次/天 |
| **周报** 周日 20:00 | 本周 ≥3天有觉察记录 | 周度觉察报告摘要 | 1次/周 |

> **注意**：服务号模板消息有月度限额（4条/用户/月免费），MVP 阶段可用客服消息接口（48小时内互动免费）+ 模板消息（关键节点）组合。

### 2.3 用户旅程地图

```
关注 → 新用户引导（3轮）→ 设置偏好 → 进入日常循环
                                          │
                    ┌─────────────────────┤
                    ▼                     ▼
              晨间觉察(~3轮)        晚间觉察(~5轮)
              7:00 推送触发         21:30 推送触发
              或用户主动发起        或用户主动发起
                    │                     │
                    └──────────┬──────────┘
                               ▼
                        数据沉淀+正反馈
                               │
                    ┌──────────┼──────────┐
                    ▼          ▼          ▼
                PERMA评分  品格优势    streak+里程碑
                    │          │          │
                    └──────────┼──────────┘
                               ▼
                        周度洞察报告
                        （周日推送）
```

## 三、核心场景对话脚本

### 场景 1：新用户引导（首次关注）

```
触发：用户关注服务号
目标：建立信任、解释产品、了解用户、完成首次觉察
轮数：~5轮
```

**Bot → 用户**
> 嗨，欢迎来到觉察助手 🌾
>
> 我是你的AI觉察伙伴。每天早晚我会陪你做一次简短的自我觉察——不是填表，不是打卡，就是聊几句。
>
> 先做个小调查：你之前有写日记或做冥想的习惯吗？

**用户 → "偶尔写写日记，冥想试过没坚持下来"**

**Bot → 用户**
> 挺正常的。冥想和觉察都不需要"坚持"，只需要"停下来看看"。
>
> 我们的方式更简单：每天早晚各聊 2-3 分钟，我会问你几个问题，你随便回答就好。
>
> 方便告诉我，你一般几点起床、几点结束一天？我会在合适的时间提醒你。

**用户 → "大概7点多起，11点左右睡"**

**Bot → 用户**
> 收到。那我们约定：
> - 🌅 早上 7:15 左右，我会问你一句早安
> - 🌙 晚上 21:30 左右，我们聊几句回顾今天
>
> 不想聊的时候直接忽略就好，没有压力。
>
> 现在，要不要试着做一次觉察？就现在，2 分钟就好。

**用户 → "好"**

**Bot → 用户**
> 👇 现在暂停一下，感受一下你此刻的状态——
>
> 1. 身体什么感觉？（紧绷？放松？累？精神？）
> 2. 脑子里在想什么？（可以随便说，一个词也行）

**用户 → "有点累，肩颈紧，脑子里在想明天的工作安排"**

**Bot → 用户**
> 你能停下来感受到肩颈紧、脑子里在转明天的事——这就是觉察。
>
> 不是什么高深的东西，就是"注意到自己现在的状态"。你刚才已经做了一次。
>
> 🌱 恭喜你，觉察第 1 天完成。
>
> 明早我会来问你早安。在此之前，如果任何时候你想聊聊，直接发消息给我就好。
>
> 晚安 🌙

### 场景 2：日常晨间觉察

```
触发：定时推送（7:15）或用户发送「早」「早安」「觉察」
目标：设今日意图，激活正念
轮数：~3轮
时长：30-60秒
```

**Bot → 用户（推送）**
> 早安 🌅
>
> 昨晚睡得怎么样？
> 今天有什么事，你希望能做到全情投入？

**用户 → "睡了6个小时，不太够。今天要赶一个方案，希望能专注写完"**

**Bot → 用户**
> 6小时确实少了点。赶方案时如果觉得困，试试起身走动5分钟再回来，比硬撑有效。
>
> 写方案这件事，今天如果只能做到一件事：在写的时候把手机扣过去。不是为了自律，就是给自己一个不被打扰的空间。
>
> 晚上我们聊聊今天感觉如何。
> 🌱 觉察第 8 天 | 本周 4/7

---

### 场景 3：日常晚间觉察

```
触发：定时推送（21:30）或用户发送「晚安」「回顾」「今天」
目标：心神合一回顾 + 情绪改写 + PERMA评分
轮数：~5轮
时长：60-120秒
```

**Bot → 用户（推送）**
> 晚上好 🌙
>
> 回顾今天：
> 1. 有几个瞬间你觉得自己完全沉浸在手头的事里？
> 2. 有没有哪个时刻，你发现自己在焦虑未来或反刍过去？

**用户 → "写方案的时候有一段挺投入的，大概一个多小时。下午开会的时候一直在想别的事，有点焦虑"**

**Bot → 用户**
> 那一个多小时的投入很珍贵——你在写什么内容的时候进入状态了？
>
> 下午开会走神的时候，脑子里在转什么？

**用户 → "写用户故事的时候很顺，感觉思路很清晰。开会的时候在想竞品会不会抢先发布，有点停不下来"**

**Bot → 用户**
> 你看到了两件事：
> 1. 做具体的事（写用户故事）时你是能投入的 ← 这是你的 E 时刻
> 2. 焦虑竞品的时候，脑子在"编故事" ← 这不是真的在解决问题
>
> 停一下，轻轻问自己：**谁在焦虑？**
> 那个"知道自己在焦虑"的东西，它本身并不焦虑。
>
> 今天你意识到自己在焦虑，还继续开完了会——这就是带着焦虑在行动。
>
> 给今天打个分吧（1-10凭直觉）：
> **P** 积极情绪 / **E** 投入心流 / **R** 人际关系 / **M** 意义感 / **A** 成就感

**用户 → "P 5  E 8  R 4  M 6  A 7"**

**Bot → 用户**
> 今天有什么值得感恩的事？

**用户 → "感恩下午有个同事主动帮我review了文档"**

**Bot → 用户**
> 今天的觉察总结：
>
> - 🔥 你的投入时刻：写用户故事（E=8，本周最高）
> - 🌊 被看见的模式：焦虑竞品→脑子编故事→你注意到了→没被带走
> - 💛 人际温暖：同事主动帮忙（R 维度）
> - 🧠 今天你经历了一次改写：焦虑还在，但你带着它行动了
>
> 明天给自己一个小改变：写方案前，花 2 分钟把"竞品焦虑"写在纸上，写完就放一边。试试看？
>
> 🌱 觉察第 8 天 | 连续 5 天 | 🌿 里程碑：发芽 → 成长（还差 1 天！）
> 本周 4/7 | P均值5.2 | E均值6.8

### 场景 4：周度洞察报告

```
触发：周日 20:00 定时推送
目标：可视化一周觉察数据，提炼模式，引导下周行动
形式：图文消息（H5 页面或长文本）
```

**Bot → 用户**

> 📊 本周觉察报告（5/11 - 5/17）
>
> ---
>
> **完成率**：晨间 6/7（86%）| 晚间 5/7（71%）
>
> **PERMA 均值**：
> ```
> P 积极情绪  ████████░░  5.4
> E 投入心流  ██████████  7.2  ⬆ 本周最高
> R 人际关系  █████░░░░░  4.8  ⬇ 需关注
> M 意义感    ███████░░░  6.1
> A 成就感    ████████░░  6.7
> ```
>
> **本周模式**：
> - 🔥 你的投入时刻集中在"写代码/写方案"——做具体产出时 E 最高
> - 🌊 焦虑模式出现 3 次，主题都是"竞品/进度"，但你每次都觉察到了
> - 💛 R 维度偏低：本周社交互动少，下周可以安排一次有质量的交流
>
> **品格优势 TOP 3**：
> 1. 🎯 毅力 — 焦虑时仍完成任务
> 2. 🔍 好奇心 — 持续觉察自己的模式
> 3. 🤝 感恩 — 3 次主动感恩他人帮助
>
> **下周小实验**：
> 你的 E 在独处创作时最高。试试下周安排 2 个上午的 90 分钟深度工作窗口（手机静音），观察 E 的变化。
>
> ---
> 🌳 连续觉察第 7 天！解锁里程碑：**成长**
>
> "觉察不是变成另一个人，是更好地成为自己。"

### 场景 5：中途触发（用户随时发消息）

```
触发：用户在任意时刻发送自由文本
目标：识别情绪/意图，提供即时觉察引导
```

**用户 → "好烦啊什么都不想做"**

**Bot → 用户**
> 停一下，感受一下"好烦"——
>
> 是身体累？还是脑子转太多？还是某件事卡住了？
> 不用分析，随便说。

**用户 → "就是刷了一下午手机，什么都没干，越刷越烦"**

**Bot → 用户**
> 刷手机→自责→越烦越刷→更自责。你看到了这个循环对吧？
>
> 现在可以做一件事：**带着"不想做"的感觉，去做最小的一步。**
>
> 你今天本来最想做的那件事是什么？不是"应该做"的，是"想做"的。

**用户 → "想写那个功能，但一直没开始"**

**Bot → 用户**
> 那就做一件事：打开编辑器，建个文件。不用写代码，就建个文件。
>
> Musk 说的：第一次做就能消除 80% 的恐惧。
>
> 做完告诉我一声就行。

## 四、技术实现要点

### 4.1 服务号能力与限制

| 能力 | 说明 | MVP方案 |
|------|------|---------|
| 被动回复 | 用户发消息后5秒内必须响应 | Webhook → 后端 → LLM → 回复 |
| 客服消息 | 48小时内用户互动过可无限发 | 日常推送走这个通道 |
| 模板消息 | 4条/用户/月（免费额度） | 留给里程碑/周报等高价值场景 |
| 菜单 | 底部3个tab | 觉察/我的/关于 |
| H5页面 | 菜单可跳外部链接 | 后期做PERMA看板 |

### 4.2 后端架构（极简MVP）

```
用户消息 → 微信服务器 → Webhook(POST) → 路由层
                                              │
                                    ┌─────────┼──────────┐
                                    ▼         ▼          ▼
                               意图识别    觉察对话    系统指令
                               (分类器)   (LLM+Skill) (菜单/关于)
                                    │         │
                                    └────┬────┘
                                         ▼
                                    数据层
                               profile + journal + streak
                                    │
                                    ▼
                              定时推送引擎
                          (晨间/晚间/周报 scheduler)
```

### 4.3 数据模型（最小集）

```json

## 五、MVP 范围定义

### In Scope（v1）

- [x] 新用户引导流程
- [x] 晨间觉察（3轮）
- [x] 晚间觉察（5轮）
- [x] 中途自由对话（情绪即时响应）
- [x] PERMA 评分采集
- [x] streak + 里程碑
- [x] 周报（文本版）
- [x] 底部菜单（觉察/我的/关于）

### Out of Scope（v2+）

- [ ] H5 看板（PERMA趋势图、品格画像）
- [ ] 小程序嵌入
- [ ] 可穿戴设备数据接入
- [ ] 社群/分享功能
- [ ] 多语言
- [ ] 付费会员
