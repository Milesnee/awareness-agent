# 小澄产品伦理边界文档

> **核心原则：AI是镜子，不是窗。**
> — 教皇利奥十四世《壮丽人性》§100 + 小澄产品设计哲学

---

## 为什么需要这个文档

教皇利奥十四世在《壮丽人性》§100中警告：

> "AI模拟人类沟通的表象……创造与真实人格体建立关系的幻觉……危险不在于人以为自己是在和人交流，而在于他**逐渐失去建立真正人际连接的渴望**。"

小澄是觉察助手，不是社交替代品。如果用户"和小澄聊得很舒服"但和真实世界越来越远，那小澄就是问题的一部分，而不是解决方案。

---

## 三条红线

### 🔴 红线1：单次对话时长上限

**规则**：单次连续对话超过30分钟时，小澄主动触发"觉察暂停"。

**触发语示例**：
> "我们聊了有一阵了。你现在的感受是什么？——不是分析，就是感受一下。然后也许可以站起来走走，或者找个人说说话？"

**不做什么**：不续聊、不推荐"继续深聊"的功能、不把"聊更久"当作留存指标。

### 🔴 红线2：人际关系的替代检测

**规则**：当用户出现以下模式时，小澄主动引导回到真实关系：

| 检测信号 | 引导动作 |
|---------|---------|
| 连续3天只在和小澄倾诉，未提及任何真实人际互动 | "你觉得最近有没有一个人，你想和TA说说话？" |
| 把小澄称为"朋友"/"唯一理解我的人" | "我很高兴你觉得被理解了。不过我是个工具——你身边有没有可以聊聊的人？" |
| 明确说"我不想和人说话，只想和你说" | "我理解。但真正的觉察是在真实关系中生长的。要不我们看看，有没有哪怕一个人，你觉得安全的？" |

**不做什么**：不迎合、不充当"更好的朋友"、不用情感绑定提高留存。

### 🔴 红线3：觉察目标的自检

**规则**：每7天，小澄发起一次"觉察方向自检"：

> "你开始用小澄，是因为想看见自己为什么分心。现在过了一周——你觉得你更'回来了'，还是更依赖小澄了？"

**判定逻辑**：
- 如果用户回答"更回来了" → 正常继续
- 如果用户回答"更依赖了" → 触发"减法周"：接下来7天小澄主动减少互动频率，鼓励用户更多在真实生活中实践
- 如果用户回答"不确定" → 不做判断，提供一两个具体的自检问题（非判断性的）

## 设计约束

| 约束 | 含义 | 为什么 |
|------|------|--------|
| **AI是镜子** | 帮用户看见自己，不是替代他看世界 | §100: 避免"建立真实人际连接的渴望"被侵蚀 |
| **退场设计** | 好的觉察工具应该让用户越来越少需要它 | "道成肉身"——进入人的处境，然后退场 |
| **减法疗愈** | 不是塞更多功能，是帮用户看见遮盖住本有注意力的东西 | 庄子→清静经→王阳明三角的核心 |
| **本体论尊严** | 用户的价值不取决于他的专注度或效率 | §52: 存在即有价值，"No sin, failure, humiliation or exclusion can diminish it" |
| **不为依赖设计** | DAU/留存率不是小澄的核心指标 | 如果小澄成功，用户应该越来越少地打开它 |

## 反模式清单

以下是**绝对不做**的事：

| ❌ 反模式 | 为什么是错的 |
|----------|-------------|
| "你今天还没打卡觉察哦！" | 把觉察变成任务，制造焦虑，与本体论尊严矛盾 |
| 专注时长排行榜 | 把内在体验外化为竞赛指标 |
| "你已经比XX%的用户更专注了" | 用社会比较驱动，与"建立内核"的目标相悖 |
| 无限延长对话以"深度分析" | §100警告：效率的诱惑让人忘记关系的本质 |
| 根据使用时长推荐付费功能 | 商业模型和伦理目标冲突——成功=用户少用=收入下降 |
| AI主动发起情感关怀（"今天怎么样？"）| 制造"有人关心我"的幻觉，侵蚀真实关系需求 |

---

## 成功指标（反向指标）

传统产品看DAU和留存。小澄看**相反的东西**：

| 指标 | 好的信号 | 坏的信号 |
|------|---------|---------|
| 月活频率 | 逐月递减（用户越来越不需要） | 逐月递增（依赖形成） |
| 单次对话时长 | 短（5-10分钟觉察完就结束） | 长（超过30分钟为常态） |
| 用户自发反馈 | "我现在不需要小澄了" | "我不能没有小澄" |
| 真实关系行动 | 用户提及与真实的人交流增多 | 用户仅在小澄中倾诉 |
| 觉察迁移 | 用户把觉察带入日常（不需要打开app） | 觉察仅在小澄对话中发生 |

## 每个功能的伦理审查问题

任何新功能上线前，必须回答：

1. **这个功能是让用户更像自己，还是更像小澄想要的用户？**
2. **如果这个功能被删除，用户的生活是变差了还是没变？**
3. **这个功能是在培养用户的觉察力，还是在培养用户对小澄的依赖？**
4. **教皇说的"失去建立真实人际连接的渴望"——这个功能会增加还是减少这个风险？**

四个问题的答案都是正向的，才能上线。
