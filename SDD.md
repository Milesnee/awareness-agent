> 创建时间：2026-05-11
> 状态：DRAFT

## 1. 需求概述

**目标**：基于 PERMA 幸福模型 + 24项品格优势理论，在飞书上为老麦搭建一个AI觉察伴侣原型，通过每日六时书对话引导，实现自我觉察的自动化追踪与可视化。

**背景**：
- 理论内核：清华大学积极心理学 + 马丁·塞利格曼 PERMA 模型
- 老麦已有六时书实践（小麦自动生成），现在要升级为结构化觉察系统
- 未来计划：独立微信服务号，多用户 SaaS 产品

**用户**：MVP 阶段仅老麦自己，验证交互体验和核心价值

### 2.2 边界条件

**做什么**：
- 飞书私聊对话（复用现有小麦连接器）
- 自然语言对话引导（不是表单/问卷式）
- 结构化数据持久化（JSON + Markdown）
- 周报生成（Markdown + 飞书文件推送）

**不做什么**：
- ❌ 不做微信服务号（MVP 阶段）
- ❌ 不做多用户/多租户
- ❌ 不做可穿戴数据联动（后续迭代）
- ❌ 不做雷达图可视化（先文字版，后续加图表）

**已知约束**：
- 模型：GLM-5-Turbo（当前默认），共情能力有限，需要精心设计 prompt
- 飞书消息频率限制：需控制推送频次，避免打扰
- 单用户原型：无需考虑并发和隔离

## 4. 理论内核

### 4.0 三位一体理论基础

觉察助手融合三大理论体系，形成完整的觉察闭环：

**① PERMA 幸福模型（塞利格曼）** — 量化框架
- P（积极情绪）/ E（投入心流）/ R（人际关系）/ M（意义感）/ A（成就感）
- 提供5维评分体系，让觉察可量化、可追踪

**② 24项品格优势（VIA）** — 行为词汇
- 智慧与知识（含**创造力、好奇心、批判性思维、好学、洞察力**）
- 勇气（含**勇敢、毅力、真诚、热情**）
- 仁爱（含**善良、爱、社会智慧**）
- 正义（含**公平、领导力、团队合作**）
- 节制（含**宽恕、谦逊、谨慎、自我调节**）
- 超越（含**美感、感恩、希望、幽默、灵性**）
- 为用户的觉察行为提供**品格层面的命名**，让模糊的"我变好了"变成"我今天调用了毅力和好奇心"

**③ 心神合一（东方心学）** — 实践方法
- 核心定义：在觉知观照下、良知指引中、正念守护里，全神贯注做好当下每一件事
- 不是简单的专注力，而是一种**活着的方式** — "砍柴即砍柴，担水即担水"
- 觉察技巧：情绪翻涌时问"谁在生气？" — 发现自己不是情绪本身，是观察情绪的那片天空
- 最高境界：不是"获得"心神合一的能力，而是"认出"自己本来就是合一的

**三位一体的协同**：
- PERMA 提供**量化刻度**（你今天状态如何？）
- 品格优势提供**行为词汇**（你调用了什么力量？）
- 心神合一提供**实践锚点**（你有哪些瞬间身心俱在？）
- 微小正反馈系统将三者串联：觉察锚点 → 品格命名 → PERMA评分 → 正反馈 → 更愿觉察

**融入对话引导的具体设计**：
- 晨间：「今天打算在哪个时刻练习心神合一？有什么事需要你全情投入？」
- 晚间：「今天有几个瞬间你做到了身心俱在？有没有哪个时刻你发现自己在焦虑未来或反刍过去？」
- 情绪觉察：「停一下，轻轻问自己：谁在XX？那个'知道自己在XX'的东西，它本身并不XX」
- 里程碑🌸绽放解锁语：「你不是学会了觉察，你发现觉察一直都在」

### 5.2 领域对齐机制：Grill-with-Docs

> 灵感来源：Matt Pocock /grill-with-docs（mattpocock/skills, 57k star）
> 核心理念：任何设计变更前，先逐个分支拷问计划，对标已有领域模型，确保一致性

觉察助手虽然不是编码项目，但同样需要**领域模型对齐**——PERMA五维度、24项品格优势、心神合一的概念之间容易产生歧义和混用。

**CONTEXT.md（领域术语表）**：
- 定义觉察对话中的所有核心术语的**精确含义和使用边界**
- 防止同一概念在不同对话轮次中被不同理解
- 例："投入" vs "心流" vs "心神合一" — 三个相关但不同的概念
- 每次对话/设计变更后，如发现术语不一致，**立即更新** CONTEXT.md

**ADR（架构决策记录）**：
- 每个重要设计决策写一条 ADR，格式：`docs/adr/NNN-title.md`
- 包含：决策背景、选项对比、最终选择、理由、影响范围
- 觉察助手的核心 ADR：
  - 001: 为什么选择 SKILL 驱动而非独立脚本
  - 002: 为什么 MVP 用 JSON 而非 SQLite
  - 003: 为什么用渐进式初始化而非问卷

**Grill 检查清单**（每次功能变更前必须通过）：
1. 新概念是否在 CONTEXT.md 中有精确定义？
2. 新功能是否与 PERMA/品格优势/心神合一理论体系一致？
3. 是否引入了与现有术语冲突的表述？
4. 对话模板是否保持了"提问多于建议"的引导风格？
5. 正反馈是否具体而非空洞？
6. 是否影响已有 ADR 的决策前提？

### 5.3 关键决策

| 决策点 | 选项A | 选项B | 选择 | 理由 |
|--------|-------|-------|------|------|
| 对话引擎 | 独立Python脚本 | OpenClaw SKILL驱动 | SKILL | 复用OpenClaw的session管理、消息路由、心跳机制 |
| PERMA评分提取 | 规则/关键词匹配 | LLM结构化输出 | LLM | 觉察对话是自然语言，规则太死板 |
| 存储格式 | SQLite | JSON文件 | JSON | MVP阶段简单直接，文件系统即可 |
| 主动推送 | 独立cron | OpenClaw heartbeat | heartbeat | 复用现有机制，统一管理 |
| 初始化方式 | 完整问卷 | 渐进式对话 | 渐进式 | 问卷体验差，3-5天对话自然建立profile |

### 5.3 核心设计哲学：微小正反馈系统

**设计微小正反馈系统，事无巨细地量化进度，每完成一个小目标就给自己一个奖励。**

这是觉察助手的灵魂。觉察本身是反人性的（需要直面自己），必须用行为心理学（操作性条件反射）让用户**愿意回来**。

**正反馈机制设计**：

1. **即时认可** — 用户每完成一次觉察对话（哪怕只回了一句话），立即给予正向回应：
   - "你今天愿意停下来觉察，这本身就很了不起"
   - "连续第3天了，你的觉察肌肉正在生长"
   - 不空洞表扬，具体点出用户做到了什么

2. **进度量化（事无巨细）** — 用户能看到自己的所有积累：
   - 📅 连续觉察天数（streak）
   - 🎯 本周完成几次晨间/晚间觉察
   - 📊 PERMA五维评分趋势（这周vs上周）
   - 💪 本周调用了哪些品格优势
   - 📝 累计觉察记录条数
   - 🏆 里程碑解锁（首次觉察、连续7天、首次识别品格优势、PERMA全绿等）

3. **小目标奖励系统** — 把大目标拆成极小的可完成单元：
   - 🌱 种子级：回复一次觉察引导（1分钟）
   - 🌿 发芽级：完成晨间+晚间双觉察（1天）
   - 🌳 成长级：连续3天觉察
   - 🎋 茂盛级：连续7天 + 生成首份周报
   - 🌸 绽放级：识别出自己的签名优势
   - 每个等级解锁时给予**具体、个性化的肯定**（不是模板）

4. **可视化进度条** — 在每次交互中自然融入进度提示：
   - 晨间：「今天是你连续觉察的第 X 天，超过了 80% 的尝试者 🌱」
   - 晚间：「本周觉察完成度 5/7，还差2天就能解锁周报啦」
   - 不强行推销，像游戏NPC一样自然提醒

5. **延迟满足 + 即时满足平衡**：
   - 即时：每次对话后的认可和鼓励（P-积极情绪）
   - 短期：连续streak带来的成就感（A-成就感）
   - 中期：周报中的成长轨迹可视化（M-意义感）
   - 长期：品格优势画像的逐渐清晰（M-意义感 + E-投入）

**与PERMA的映射**：
- P（积极情绪）← 每次小奖励带来的快乐
- E（投入/心流）← 连续streak带来的心流状态
- R（人际关系）← 觉察中提到的关系改善
- M（意义感）← 进度可视化和周报中看到的成长轨迹
- A（成就感）← 里程碑解锁 + 量化数据证明"我在变好"

### 5.4 Prompt 设计原则
- 觉察助手人格：温暖但不油腻、有深度但不学术、像一个靠谱的老朋友
- 不说教、不评判、不灌鸡汤
- 提问多于建议，引导用户自己发现
- 每次对话控制在3-5轮，不拖沓
- **必须融入正反馈**：对话结尾带进度/连续天数/具体认可
- **融入心神合一锚点**：晨间设意图、晚间回顾身心俱在时刻、情绪觉察用"谁在XX"技巧
- 关键：从用户回复中**提取结构化数据**（PERMA评分、品格优势、心神合一瞬间、关键事件）

### 5.5 对话模板设计

**晨间觉察引导（~3轮）**：
```
Round 1: 早安，[昵称]。
  昨晚睡得怎么样？（1-5分或自由描述）
  今天有什么事需要你全情投入？打算在哪个时刻练习心神合一？

Round 2: [根据回复，具体认可+引导PERMA某维度]
  例：「你提到今天要深度写代码，这是个很好的E（投入）时刻」

Round 3: [设一个小意图]
  「那今天给自己一个小目标：写代码时如果发现自己走神了，轻轻拉回来就好。
   不评判，只是拉回来。晚上我们聊聊感觉如何？」
  + 进度提示（streak/本周完成度）
```

**晚间觉察引导（~5轮）**：
```
Round 1: 晚上好，[昵称]。回顾今天：
  1. 今天有几个瞬间你做到了身心俱在？
  2. 有没有哪个时刻，你发现自己在焦虑未来或反刍过去？

Round 2: [情绪觉察] 
  今天最强烈的情绪是什么？
  如果是负面情绪，试着问自己：「谁在XX？」

Round 3: [品格优势识别]
  今天你调用了什么品格优势？（好奇心？毅力？自我调节？）
  [如果用户说不出来，引导：「面对XX时你没有放弃，这本身就是毅力」]

Round 4: [PERMA评分提取]
  「如果今天给这5个维度打个分（1-10）：
   P 积极情绪 / E 投入心流 / R 人际关系 / M 意义感 / A 成就感」

Round 5: [感恩 + 正反馈]
  「今天有什么值得感恩的事？」
  + 连续天数/streak + 个性化肯定 + 进度提示
```

- 觉察助手人格：温暖但不油腻、有深度但不学术、像一个靠谱的老朋友
- 不说教、不评判、不灌鸡汤
- 提问多于建议，引导用户自己发现
- 每次对话控制在3-5轮，不拖沓
- **必须融入正反馈**：对话结尾带进度/连续天数/具体认可
- 关键：从用户回复中**提取结构化数据**（PERMA评分、品格优势、关键事件），不是纯聊天

## 7. 风险与不确定性

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| GLM-5 共情能力不足 | 对话体验差，用户不想聊 | 中 | 精心设计prompt，增加few-shot示例，后续可切换Claude |
| PERMA评分提取不准 | 数据质量差，报告无意义 | 中 | LLM结构化输出 + 人工校验（老麦自己看） |
| 推送打扰老麦工作 | 体验负面 | 低 | 飞书消息不强制，NO_REPLY机制兜底 |
| 六时书太长/太频繁 | 用户疲劳 | 中 | 控制在3-5轮，晚间总结不超过5分钟 |

## 8. 验收标准
