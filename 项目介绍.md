
## 项目名称
**Hermes 觉察助手（Awareness Guide）**


## 项目简介（300字）

Hermes 是一个 AI 驱动的每日觉察引导系统。基于 PERMA 幸福模型（Seligman, 2011）、心流理论（Csikszentmihalyi）、24 项品格优势框架（Peterson & Seligman），通过飞书 Bot 主动推送晨间/晚间结构化觉察对话，帮助用户建立自我觉察习惯，追踪情绪与行为模式。

核心差异化：不只做"聊天"。Hermes 构建了一个四层数据体系——叙事层（用户说了什么）→ 行为层（系统自动收集用户做了什么）→ 知识层（提取用户的决策/追踪/困扰）→ 行动层（转化为可追踪的 Task 并闭环跟进）。周度生成 AAR（行动后复盘）洞察报告，把流水账变成可验证的个人成长飞轮。

对标 Anthropic 官方 14 个 Skill 设计模式全覆盖。纯 Python 标准库实现，零外部依赖。多租户数据架构，数据层与 AI 底座解耦，支持随时迁移到微信服务号独立部署。

## 项目亮点

### 1. 技术架构 —— 14 个 Anthropic 官方 Skill 模式全覆盖
- **渐进式披露**：SKILL.md 370 行入口，7 个 references 按需加载，Token 预算可控
- **Explain-the-Why**：9 条核心规则全部附带"为什么"，LLM 在边界情况能自主判断
- **已知陷阱（Edge Cases）**：基于真实用户反馈持续积累的突发情况处理
- **Template Scaffold**：知识碎片提取、周洞察输出都有严格模板约束
- **多租户数据隔离**：`data/users/{uid}/` 独立空间，隐私合规就绪

### 2. 产品架构 —— 四层数据飞轮
```
叙事层 → 行为层 → 知识层 → 行动层
（对话）  （日志）  （碎片）  （Task闭环）
    ↓         ↓        ↓        ↓
  "说了什么""做了什么""在想什么""要做什么"
```
周度聚合四条线 → AAR 偏差分析 → 可验证假设 → 实验方向

### 3. AI 驱动全流程
- 对话引擎：SKILL 驱动，复刻 Anthropic Claude Code Agent Harness 8 步循环
- 自动化：晨间 08:00 / 晚间 21:30 定时推送 + 活动收集 + 知识碎片提取 + Task 闭环
- 周洞察：AAR（行动后复盘）+ 科学方法（假设-验证），自动聚合生成

### 4. 零成本启动
- Python 标准库，零外部依赖
- JSON 文件存储，无需数据库
- 复用现有飞书 Bot，零渠道成本
