## 1. 需求概述

**目标**：基于庄子→清静经→王阳明理论三角 + 减法疗愈框架，在飞书上搭建一个 AI 觉察引导系统，帮助用户逐步找到并建立稳定的内核。

**背景**：
- 理论内核：庄子（为什么要放下）→ 清静经（怎么放下）→ 王阳明（放下后怎么行动）
- 底层哲学：减法疗愈——不是塞更多道理，是帮用户看见遮盖住本有注意力的东西
- 终极定义：帮用户逐步找到并建立稳定的内核。不是管理时间，不是做计划，是建立内核。
- 口号：向外看的人在做梦，向内看的人在觉醒。 ——荣格
- 前期以对话式觉察引导为主，不依赖硬件
- 后期可整合可穿戴设备数据（情绪感知辅助）

**用户**：MVP 阶段仅老麦自己，验证交互体验和核心价值。客户发现阶段面向泛 ADHD 人群。

### 2.2 边界条件

**做什么**：
- 飞书私聊对话（复用现有小麦连接器）
- 自然语言对话引导（减法式：提问多于建议，帮用户自己看见）
- 结构化数据持久化（JSON + Markdown）
- 周报生成（Markdown + 飞书文件推送）

**不做什么**：
- ❌ 不教冥想放松（不是冥想 App）
- ❌ 不靠番茄钟强制（不是时间管理工具）
- ❌ 不做微信服务号（MVP 阶段）
- ❌ 不做多用户/多租户
- ❌ 不做可穿戴数据联动（后续迭代）
- ❌ 不做雷达图可视化（先文字版，后续加图表）

**已知约束**：
- 模型：GLM-5-Turbo（当前默认），共情能力有限，需要精心设计 prompt
- 飞书消息频率限制：需控制推送频次，避免打扰
- 单用户原型：无需考虑并发和隔离

## 4. 理论内核

### 4.0 理论三角：庄子→清静经→王阳明

小澄的底层哲学是**减法疗愈**——不是塞更多道理，是帮用户看见遮盖住本有注意力的东西。

**① 庄子——为什么要放下**
- 核心概念：心斋、坐忘、齐物
- "堕肢体，黜聪明，离形去知，同于大通"（《大宗师》）
- 放下的不是追求本身，是遮蔽你本有觉知的东西
- 当你在刷手机时，不是"你在刷手机"，是"有东西遮住了你本来就知道该做的事"
- 应用：「你觉得是什么把你拉走了？不是手机本身，是手机后面的那个东西」

**② 清静经——怎么放下**
- "人能常清静，天地悉皆归"
- "大道无形，生育天地；大道无情，运行日月"
- 静下来的方法：不是用力去静，是看见躁动本身就会静下来
- "上士无争，下士好争；上德不德，下德执德"
- 应用：「不用刻意让自己专注，先看看那个让你分心的东西是什么，看见它就够了」

**③ 王阳明——放下后怎么行动**
- "知行合一"——知道了就自然去做，不做等于不知道
- "致良知"——你本来就知道该做什么，只是被遮蔽了
- "你未看此花时，此花与汝心同归于寂；你来看此花时，则此花颜色一时明白起来"
- 有了内核才能物来顺应、不迎不拒
- 应用：「你本来就知道答案。如果你不知道，说明有什么东西遮住了你」

**三者的协同关系**：
- 庄子告诉你**为什么要放下**（遮蔽了本有的觉知）
- 清静经告诉你**怎么放下**（看见躁动本身就会静下来）
- 王阳明告诉你**放下后怎么行动**（致良知，知行合一）
- 最终目标：**建立稳定的内核**——不是学会了什么能力，是认出了自己本来就是怎样的

**减法疗愈的核心逻辑**：
1. 看见分心的因果链（不是"我在分心"，是"什么遮住了我"）
2. 放下遮蔽（不是用力放下，是看见就够了）
3. 内核自然浮现（它一直在那里，只是被遮住了）
4. 行动自然发生（致良知，不需要意志力硬撑）

**融入对话引导的具体设计**：
- 晨间：「今天有什么事在等你？你打算在哪个时刻建立自己的内核锚点？」
- 晚间：「今天你被什么拉走了？能看见那条因果链吗？——从什么开始，经过什么，到什么结果」
- 觉察体验：「你刚才描述的那个时刻，你有没有感觉到，放下之后其实是轻松的？」
- 里程碑🌸绽放解锁语：「你不需要学会觉察，你发现觉察一直都在」

### 5.2 领域对齐机制：Grill-with-Docs

> 灵感来源：Matt Pocock /grill-with-docs（mattpocock/skills, 57k star）
> 核心理念：任何设计变更前，先逐个分支拷问计划，对标已有领域模型，确保一致性

**CONTEXT.md（领域术语表）**：
- 定义觉察对话中的所有核心术语的**精确含义和使用边界**
- 防止同一概念在不同对话轮次中被不同理解
- 例："放下" vs "逃避" vs "压抑" — 三个容易混淆的概念
- 每次对话/设计变更后，如发现术语不一致，**立即更新** CONTEXT.md

**ADR（架构决策记录）**：
- 每个重要设计决策写一条 ADR，格式：`docs/adr/NNN-title.md`
- 已有 ADR：
  - 001: 为什么选择 SKILL 驱动而非独立脚本
  - 002: 为什么 MVP 用 JSON 而非 SQLite
  - 003: 为什么用渐进式初始化而非问卷

**Grill 检查清单**（每次功能变更前必须通过）：
1. 新概念是否在 CONTEXT.md 中有精确定义？
2. 新功能是否与减法疗愈框架一致？
3. 是否引入了与理论三角冲突的表述？
4. 对话模板是否保持了"提问多于建议"的减法式引导？
5. 正反馈是否具体而非空洞？
6. 是否影响已有 ADR 的决策前提？

### 5.3 关键决策

| 决策点 | 选项A | 选项B | 选择 | 理由 |
|--------|-------|-------|------|------|
| 对话引擎 | 独立Python脚本 | OpenClaw SKILL驱动 | SKILL | 复用OpenClaw的session管理、消息路由、心跳机制 |
| 存储格式 | SQLite | JSON文件 | JSON | MVP阶段简单直接，文件系统即可 |
| 主动推送 | 独立cron | OpenClaw heartbeat | heartbeat | 复用现有机制，统一管理 |
| 初始化方式 | 完整问卷 | 渐进式对话 | 渐进式 | 问卷体验差，3-5天对话自然建立profile |
| 理论框架 | PERMA+品格优势 | 庄子→清静经→王阳明 | 理论三角 | 减法疗愈更符合"建立内核"目标，差异化更强 |

### 5.3 核心设计哲学：微小正反馈系统

**设计微小正反馈系统，事无巨细地量化进度，每完成一个小目标就给自己一个奖励。**

觉察本身是反人性的（需要直面自己），必须用行为心理学让用户**愿意回来**。

**正反馈机制设计**：

1. **即时认可** — 用户每完成一次觉察对话（哪怕只回了一句话），立即给予正向回应：
   - "你今天愿意停下来觉察，这本身就很了不起"
   - "连续第3天了，你的觉察肌肉正在生长"
   - 不空洞表扬，具体点出用户做到了什么

2. **进度量化（事无巨细）**：
   - 📅 连续觉察天数（streak）
   - 🎯 本周完成几次觉察
   - 📊 内核稳定性趋势
   - 📝 累计觉察记录条数
   - 🏆 里程碑解锁

3. **小目标奖励系统**：
   - 🌱 种子级：回复一次觉察引导（1分钟）
   - 🌿 发芽级：完成晨间+晚间双觉察（1天）
   - 🌳 成长级：连续3天觉察
   - 🎋 茂盛级：连续7天 + 生成首份周报
   - 🌸 绽放级：第一次独立"看见"分心因果链
   - 每个等级解锁时给予**具体、个性化的肯定**

4. **延迟满足 + 即时满足平衡**：
   - 即时：每次对话后的认可和鼓励
   - 短期：连续streak带来的成就感
   - 中期：周报中的内核稳定性趋势
   - 长期：分心模式的逐渐清晰，内核的逐渐稳定

### 5.4 Prompt 设计原则
- 小澄人格：温暖但不油腻、有深度但不学术、像一个靠谱的老朋友
- 不说教、不评判、不灌鸡汤
- 提问多于建议，帮用户自己看见（减法式引导）
- 每次对话控制在3-5轮，不拖沓
- **必须融入正反馈**：对话结尾带进度/连续天数/具体认可
- **围绕"建立内核"**：晨间设锚点、晚间回顾因果链
- 关键：从用户回复中**提取结构化数据**（内核稳定性、分心模式、觉察体验）

### 5.5 对话模板设计

**晨间觉察引导（~3轮）**：
```
Round 1: 早安，[昵称]。
  昨晚睡得怎么样？
  今天有什么事在等你？打算在哪个时刻建立自己的内核锚点？

Round 2: [根据回复，具体认可+引导深入]
  例：「你提到今天要深度写代码——那个时刻，你打算怎么让自己全情投入？」
  例：「你说今天比较乱——是什么让你觉得乱？能看见那条线吗？」

Round 3: [设一个小意图]
  「那今天给自己一个小目标：[具体锚点]。
   如果发现自己被拉走了，不用评判，轻轻问自己：是什么在拉我？
   晚上我们聊聊。」
  + 进度提示（streak/本周完成度）
```

**晚间觉察引导（~5轮）**：
```
Round 1: 晚上好，[昵称]。回顾今天：
  1. 今天你被什么拉走了？能看见那条因果链吗？
  2. 有没有哪个时刻，你觉得自己的内核是稳定的？

Round 2: [因果链深入]
  从什么开始的？中间经过什么？最后是什么结果？
  [帮用户把模糊的"我又分心了"变成具体的因果链]

Round 3: [觉察体验]
  当你看见那条因果链的时候，有什么感觉？
  是不是放下之后反而轻松了？

Round 4: [内核稳定性自评]
  如果给今天的内核稳定性打个分（1-10），你会给几分？
  什么让它没到10分？什么让它没到1分？

Round 5: [感恩 + 正反馈]
  「今天有什么值得感恩的事？」
  + 连续天数/streak + 个性化肯定 + 进度提示
```

## 7. 风险与不确定性

| 风险 | 影响 | 概率 | 缓解措施 |
